📚 Deployment & Ads Complete Guide

For developers: AdMob setup, production builds, and app store submission

📖 This guide covers: AdMob account setup, creating ad units, configuring production builds with EAS, submitting to App Store Connect (iOS) and Google Play Console (Android), monitoring performance, and troubleshooting common issues.

1️⃣ AdMob Setup

Create AdMob Account

  1. Go to AdMob Console
  2. Sign in with Google account
  3. Click Get Started
  4. Accept terms and fill business details

Register Your Apps

For iOS:

  1. Apps → Add App → Select iOS
  2. Name: "The Hindu Calendar - iOS"
  3. Bundle ID: com.hinducalendar.app
  4. Note your App ID

For Android:

  1. Apps → Add App → Select Android
  2. Name: "The Hindu Calendar - Android"
  3. Package: com.dineshruhela.hinducalendar
  4. Note your App ID

Create Ad Units

TypeNamePlatform
BannerHome Screen BanneriOS + Android
InterstitialScreen TransitionsiOS + Android
App OpenApp LaunchiOS + Android
Rewarded (optional)Festival InfoiOS + Android

For each ad unit, copy the Ad Unit ID and add it to your code.

2️⃣ App Configuration

Update app.json

{
  "ios": {
    "bundleIdentifier": "com.hinducalendar.app",
    "config": {
      "googleMobileAdsAppId": "ca-app-pub-XXXXXXXX~YYYYYYYY"
    }
  },
  "android": {
    "package": "com.dineshruhela.hinducalendar",
    "config": {
      "googleMobileAdsAppId": "ca-app-pub-XXXXXXXX~ZZZZZZZZ"
    }
  }
}

Update src/utils/ads.ts

Ensure all Ad Unit IDs are set to production values (no XXXXXXX).

3️⃣ Production Build Process

Build iOS

eas build --platform ios --profile production --non-interactive

Build Android

eas build --platform android --profile production --non-interactive

Monitor Status

eas build:list --limit 2
💡 Builds typically take 30–60 minutes. You'll receive email notifications when complete.

4️⃣ iOS App Store Submission

Complete App Privacy

  1. App Store Connect → App Privacy → Edit
  2. Mark data used by ads:
    • ✅ Identifiers → Purpose: Advertising
    • ✅ Usage Data → Purpose: App Functionality
  3. Save

Submit for Review

eas submit --platform ios --latest --non-interactive

Or manually:

  1. App Store Connect → Build section
  2. Select your EAS build
  3. Add release notes
  4. Click Submit for Review
⏱️ Expected time: 24–48 hours for review.

5️⃣ Android Google Play Submission

Complete Data Safety

  1. Google Play Console → Data safety
  2. Mark data collected:
    • ✅ Identifiers (Ad ID)
    • ✅ Usage Data
  3. Save

Create Release

  1. Release → Production → Create new release
  2. Upload .aab file from EAS
  3. Add release notes
  4. Review and save
  5. Click Roll out to production
⚡ Expected time: 1–3 hours for review (usually faster than iOS).

6️⃣ Post-Launch Monitoring

After 24 Hours

  1. AdMob Console → Overview
  2. Check:
    • Impressions: Should be > 0
    • Revenue: Initial earnings
    • Fill rate: % of requests fulfilled
📊 Expect: Low activity initially. Ramps up as more users install. By week 1, aim for 50–200 impressions/day for a new app.

7️⃣ Troubleshooting

Ads Not Showing

  • Check: Bundle IDs match AdMob registration
  • Wait: AdMob takes 24–48 hours to serve ads for new apps
  • Rebuild: eas build --platform ios --clear-cache --profile production

Build Fails

  • Log in: eas login
  • Clear cache: --clear-cache flag
  • Check logs: Click build ID in console

App Store Rejection

  • Review rejection email for specific reasons
  • Fix issue (usually ad placement or data disclosure)
  • Increment version, rebuild, resubmit
